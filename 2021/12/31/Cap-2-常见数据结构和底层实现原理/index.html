切片扩容

使用append向slice追加元素时，如果slice空间不足，会触发slice扩容
实际操作：重新分配一块更大的内存，将old slice拷贝到new slice，然后返回new slice，最后将数据追加到new slice

扩容规则

1.cap < 1024,新slice的容量将扩大为原来的2倍
2.cap > 1024,新slice的容量扩大为原来的1.25倍

3. map 映射

map底层用Hash表实现

3.1 特性

初始化

增删改查

Tips
  • 在修改操作中，如果 key 不存在，就转为了插入操作
  • delete() 是 map 的内置函数，且没有返回值，在 map 为 nil 或者键不存在时，delete() 不报错，相当于空操作
  • map 只能使用 len() 返回 map 内键值对的个数

并发读写

map 操作不是原子的，意味着，map 不支持并发读写，不是线程安全的，多个线程同时操作 map 可能会产生读写冲突

Go1.9 中引入了标准库 sync 下的 Map ， sync 包下面的 map 支持并发读写

相关链接: Go 1.9 sync Map 源码阅读笔记_衣舞晨风-CSDN博客

空map

指的是，未初始化的map，值为nil
向值为nil的map添加元素会触发panic
查询，删除空 map 不会报错
